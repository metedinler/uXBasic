# UBASIC -> Windows 11 + FreeBASIC Sinif (CLASS) Plani

## 1) Karar Ozeti
Bu planin amaci sistemi patlatmadan ilerlemektir.
Temel ilke: once mevcut davranisi kilitle, sonra yeni ozellik ekle, en son tasima yap.

## 2) Hedef
- Mevcut dilin uyumlulugunu korumak.
- Windows 11 terminal odakli calisma duzeni kurmak.
- Dile `CLASS ... END CLASS` yapisini eklemek.
- Derleyici cekirdegini adim adim FreeBASIC'e tasimak.

## 3) Kapsam Disi (Ilk Surumde YOK)
- Kalitim (inheritance)
- Arayuz/soyut sinif
- Generic (tip parametresi)
- Istisna sistemi
- Otomatik cop toplayici
- Akilli is parcacigi havuzu

Not: Bunlari ilk surume koymak riski cok artirir.

## 4) Neden Bu Sira?
Yanlis sira: "once buyuk yeniden yazim" -> cok yuksek kirilim riski.
Dogru sira: "davranis kilidi -> sinif ozelligi -> kontrollu tasima".

## 5) Mimari Yaklasim (Patlatmayan)
- Eski derleyici bir sure "dogruluk referansi" olarak kalacak.
- Yeni kod, eski kodla ayni girdiye benzer cikti verene kadar paralel kosacak.
- `CLASS` ilk asamada "sekersoz dizim" (syntax sugar) olarak eklenecek:
  - Ic modelde mevcut `TYPE` + `Type::Method` mekanigina indirgenecek.
- Ozellik bayragi olacak:
  - `%%FEATURE_CLASS ON/OFF` (varsayilan OFF)
  - OFF iken eski davranis birebir korunacak.

## 6) Sinif Ozellikleri Icin MVP Sozlesmesi

### 6.1 Yeni Anahtar Kelimeler
- `CLASS`
- `END CLASS`
- `PUBLIC`
- `PRIVATE`

### 6.2 Ilk Surum Davranisi
- Alan tanimi: `name AS TYPE`
- Yontem tanimi: mevcut `SUB ClassName::Method` ve `FUNCTION ClassName::Method`
- Erisim denetimi: derleme zamaninda kontrol (PUBLIC/PRIVATE)
- Nesne olusturma: mevcut `DIM obj AS ClassName`
- Yontem cagrisi: mevcut cagri bicimi korunur (mevcut dile uyumlu)

### 6.3 Ilk Surumde Ertelenecekler
- Kurucu/Yikici (constructor/destructor)
- Kalitim
- Sanal yontem
- Asiri yukleme (overload)

## 7) Is Parcacigi (THREAD) Konusu Icin Duru Plan
`THREAD ... END THREAD` fikri dogru ama CLASS ile ayni anda acmak riskli.
Bu nedenle faz-2 sonrasina ertelenir.

Neden?
- Paylasilan veri korunumu
- Kilit kurallari
- Bekleme/zaman asimi
- Hata yayilimi

Karar:
- Bu plan dosyasinda THREAD tasarimi acilmiyor.
- Once CLASS stabil olunca ayri plan dosyasi acilacak.

## 8) Asama Asama Yol Haritasi

## Faz 0 - Guvenli Baslangic (1 hafta)
Amac: geri donulebilir zemin kurmak.
- Kodun birebir kopyasi alinacak.
- Ayrik dal (branch) acilacak.
- Eski derleyici ile referans test seti olusturulacak.
- Komut haritasi tablosu cikartilacak (desteklenir / donusturulur / ertelenir).

Cikti:
- `baseline/` referans dosyalari
- "kirilmazlik" kontrol listesi

Kabul Kriteri:
- Referans testleri tekrarlandiginda ayni sonuc.

## Faz 1 - Sinif Sozdizimi Ekleme (1-2 hafta)
Amac: parser'a CLASS blogunu sokmak, eski kodu bozmamak.
- `CLASS`, `END CLASS`, `PUBLIC`, `PRIVATE` tokenlari eklenecek.
- Sinif tablosu (class table) olusturulacak.
- Cift isim, cift alan, cift yontem kontrolu eklenecek.
- Hata mesajlari netlestirilecek.

Cikti:
- Sinif tanimi okunabiliyor.
- Hata mesajlari anlasilir.

Kabul Kriteri:
- Eski programlar davranis degistirmeden derlenir.
- Yeni CLASS ornekleri parser seviyesinde gecer.

## Faz 2 - Sinifin Ic Temsili ve Donusum (1-2 hafta)
Amac: CLASS'i mevcut mekanige indirgemek.
- `CLASS` icerigi ic modelde `TYPE` tabanina map edilir.
- Yontem baglantisi mevcut `Type::Method` akisina baglanir.
- PUBLIC/PRIVATE derleme-zamani kontrolu eklenir.

Cikti:
- "Class as Type" donusumu calisir.

Kabul Kriteri:
- En az 20 sinif test ornegi gecer.
- Erisim ihlallerinde dogru hata kodu doner.

## Faz 3 - FreeBASIC Cekirdek Tasima (2-3 hafta)
Amac: ana yuruyen yolu FreeBASIC'e almak.
- Asama asama tasima: once token okuma -> sonra parser -> sonra semantik kontrol.
- Eski derleyici ciktilariyla satir/satir karsilastirma otomasyonu.
- Eski kodla uyumsuzluk cikarsa once uyumluluk, sonra temizlik.

Cikti:
- FreeBASIC tabanli ilk derleyici cekirdegi.

Kabul Kriteri:
- Referans testlerin >= %95'i ayni sonuc.
- Kalan farklar dokumante ve kabul edilmis.

## Faz 4 - Windows 11 Terminal Odagi (1 hafta)
Amac: terminalde stabil calisma.
- Ekran komutlari icin terminale uygun cikti duzeni.
- DOS'a bagli komutlar icin durum siniflandirma:
  - desteklenir
  - kisitli destek
  - destek disi
- Kullaniciya acik uyumluluk listesi.

Cikti:
- Windows 11 terminalde calisan surum.

Kabul Kriteri:
- Ornek set terminalde hatasiz calisir.

## Faz 5 - Sertlestirme ve Surumleme (1 hafta)
Amac: uretime hazir hale getirmek.
- Regresyon testleri
- Performans kontrolu
- Dokumantasyon
- Surum notu

Cikti:
- `v0-class-mvp` etiketi
- Geri donus (rollback) plani

Kabul Kriteri:
- Kritik hata 0
- Orta seviye hata <= 3

## 9) Riskler ve Onlemler

### Risk A: Eski kodlar bozulur
Onlem:
- Ozellik bayragi varsayilan OFF
- Her fazda regresyon testi

### Risk B: Parser karmasikligi artar
Onlem:
- Sinifi ilk etapta sade tut
- Kalitim ve constructor ertelensin

### Risk C: FreeBASIC tasimada farkli davranis
Onlem:
- Eski derleyici referans kabul edilir
- Cikti karsilastirma otomasyonu zorunlu

### Risk D: Thread hedefi erken acilirsa kirilim
Onlem:
- Thread ayri faz/ayri plan
- CLASS stabil olmadan acilmaz

## 10) Teknik Borc ve "Olmazsa Olmaz" Kurallar
- Kuralsiz buyuk refactor YASAK.
- Tek fazda birden fazla buyuk degisiklik YASAK.
- Testsiz birlestirme YASAK.
- Geri donus yolu olmayan degisiklik YASAK.

## 11) Komut Haritalama Is Paketi (Sinif Odagi)
Bu paket Faz 0 sonunda dosyalanacak:
- CLASS -> TYPE tabanli ic temsile donusum
- END CLASS -> sinif blogu kapama
- PUBLIC/PRIVATE -> erisim kurali tablosu
- Type::Method baglantisi -> korunacak
- DIM obj AS ClassName -> korunacak

## 12) Basari Olcumleri
- Uyumluluk: referans test gecis orani
- Kararlilik: kritik hata sayisi
- Tasinabilirlik: Windows 11 terminalde ornek set basarisi
- Gelistirici verimi: yeni hata cozum suresi

## 13) Uygulama Takvimi (Toplam 7-10 hafta)
- Faz 0: 1 hafta
- Faz 1: 1-2 hafta
- Faz 2: 1-2 hafta
- Faz 3: 2-3 hafta
- Faz 4: 1 hafta
- Faz 5: 1 hafta

## 14) Bu Planda Dogru Olanlar / Yanlis Olanlar
Dogru:
- Komut karsiliklarini haritalamak
- Mevcut sistemi cok bozmadan ilerlemek
- FreeBASIC ile devam etmek

Duzeltilmesi gereken:
- CLASS ve THREAD'i ayni anda acmak (riskli)
- "tek adimda modern cok islemciliye gecis" beklentisi (gercekci degil)

## 15) Son Karar
Bu plan "kontrollu buyume" planidir.
Ilk hedef: CLASS MVP + Windows 11 terminal uyumlulugu.
Ikinci hedef: THREAD tasarimini ayri planla acmak.

## EK-1) Append-Only: Cok Ajanli Komut ve Dosya Haritalama

Bu ek bolum sadece sona eklenmistir. Onceki plan bolumlerine dokunulmamistir.

### EK-1.1) Cok Ajanli Tarama Ozeti
- Ajan-1: README ve rapor icinden komut sozlesmeleri (amac, girdi, cikti, hata) cikardi.
- Ajan-2: SOURCE katmaninda komut -> handler -> asm uretim yonlendirmesini cikardi.
- Ajan-3: AINCLUDE katmaninda dosya bazli runtime haritasi ve 32-bit varsayimlarini cikardi.

### EK-1.2) 32-bit Assembler Destegi Icin Zorunlu Kurallar
- Kod uretimi EAX/EBX/ECX/EDX/ESI/EDI 32-bit kayit duzeni ile uyumlu kalacak.
- Cagri modeli 32-bit duzende tutulacak; 64-bit zorlamasi bu fazda YOK.
- DOS kolunda INT 21h/31h/10h/1Ah tabanli yol korunacak, Win11 hedefinde bu komutlar siniflandirilacak.
- Win32 kolunda API tabanli yol (CreateFileA, ReadFile, WriteFile, SetFilePointer, MessageBoxA, TextOutA) korunacak.
- Dizi/dosya/bellek indisleri DWORD mantiginda ele alinacak.

## EK-3) Append-Only: INPUT, PRINT/INPUT Ayiraclari ve INKEY Fonksiyon Genisletmesi

### EK-3.1) Durum Tespiti

### EK-3.2) Yeni Komut: INPUT (MVP)

#### Sozdizimi

- INPUT #file_no, var1[, var2, ...]  (dosya kaynakli varyant)

#### Derleyici Sozlesmesi

- Cikti: parser dugumu + runtime cagrisi.
- Hata:

  - degisken yerine ifade verilmesi
  - dosya numarasi gecersiz
  - tur donusum hatasi

#### 32-bit Uretim Yolu
- Win32 terminal: satir okuma + token ayristirma runtime rutini.
- DOS yolu: mevcut klavye/dosya altyapisina map.

### EK-3.3) PRINT ve INPUT Icin `;` ve `,` Destegi

#### PRINT Davranisi
- `;` : ayni satir devam, otomatik yeni satir YOK.
- `,` : zone/tab mantigi (QBasic benzeri), bir sonraki yazim alanina ilerleme.

#### INPUT Davranisi (Prompt Kismi)
- `INPUT "Ad"; adMetni` : prompt yaz, ayni satirda veri bekle.
- `INPUT "Ad", adMetni` : prompt yaz, ayirac kuraliyla bekle (uyumluluk modu).

### EK-3.4) INKEY Fonksiyon Tasarimi (MVP)

#### Geriye Donuk Uyumluluk
- Eski kod uyumluluk modunda INKEY$ esdegeri map edilebilir, yeni dilde ana fonksiyon INKEY'dir.

#### Yeni Fonksiyon
- `alinanTus = INKEY(flags)`
- Opsiyonel durum alma:
  - `alinanTus = INKEY(flags, BYREF state)`

#### Neden 32-bit Durum Alani?
- 16-bit, istenen kategori ve durumlar icin dar kalir.
- 32-bit durum alani daha guvenli ve genisletilebilir.

#### Onerilen Bit Maskeleri (flags)
- KF_ANY = &H00000001
- KF_LETTER = &H00000002
- KF_DIGIT = &H00000004
- KF_FUNCTION = &H00000008
- KF_NUMPAD = &H00000010
- KF_SPACE = &H00000020
- KF_ENTER = &H00000040
- KF_SPECIAL = &H00000080

#### Onerilen Durum Bitleri (state)
- KS_SHIFT = &H00000001
- KS_CTRL = &H00000002
- KS_ALT = &H00000004
- KS_ENTER = &H00000008
- KS_SPACE = &H00000010
- KS_NUMPAD = &H00000020
- KS_FUNCTION = &H00000040
- KS_EXTENDED = &H00000080

#### Donus Degeri (alinanTus)
- Alt 8 bit: ascii (varsa)
- Sonraki 8 bit: scancode (varsa)
- Ust bitler: rezerv

#### Hata ve Sinir Durumlari
- Desteklenmeyen tus sinifi istenirse 0 don.
- Uygun tus yoksa 0 don (non-blocking).
- Blocking varyant faz-2'de degerlendirilir.

#### Test Iskeleti
- TST-INKEYF-001: KF_LETTER ile harf tusu
- TST-INKEYF-002: KF_DIGIT ile rakam tusu
- TST-INKEYF-003: KF_FUNCTION ile F1
- TST-INKEYF-004: KF_SPACE/KF_ENTER
- TST-INKEYF-005: CTRL/ALT/SHIFT durum bitleri
- TST-INKEYF-006: Filtre disi tus -> 0

### EK-7.3) INLINE Modeli (ASM_SUB/ASM_FUNCTION Yerine)

Eski model:
- `ASM_SUB ... END ASM_SUB`
- `ASM_FUNCTION ... END ASM_FUNCTION`
- `_ASM ... _END ASM`

Yeni zorunlu model:
- `INLINE(language, programId, kind, params)`
- `END INLINE`

Ornek:
`INLINE("x86_32", "nasm", "sub", "a:I32,b:I32")`
`  mov eax, [ebp+8]`
`END INLINE`

Kurallar:
- `_` ile baslayan komut adlari kullanilmaz.
- INLINE blogu SUB/FUNCTION govdesinde veya tek basina komut olarak yer alabilir.
- `kind`: `sub` veya `func`.

### EK-7.5) `_` Ile Baslayan Komutlarin Kaldirilmasi

- `_SETSTRINGSIZE`
- `_ASM`
- `_END ASM`

Yerine:
- `SETSTRINGSIZE n`
- `INLINE(...) ... END INLINE`

### EK-7.6) TIMER Genisletmesi

`TIMER` tek bir saniye donen yapi olmaktan cikarilir.

Yeni syntax:
- `TIMER()`
- `TIMER(unit)`
- `TIMER(startTick, endTick, unit)`

Desteklenen unit degerleri:
- `ns`, `us`, `ms`, `s`, `min`, `h`, `day`, `year`

Donus tipi:
- Varsayilan `I64` (ham tick/fark)
- Istenirse `F64` (donusturulmus sure)

Not:
- `ns/us` hassasiyeti isletim sistemi zamanlayici sinirlarina baglidir.
- `year` takvim degil, sure donusum birimi olarak ele alinir.
